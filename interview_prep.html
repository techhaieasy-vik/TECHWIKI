☸️ Azure Kubernetes Service (AKS)
Basic What is the difference between system and user node pools in AKS?
System node pools host critical system pods (CoreDNS, metrics-server, tunnel-front). User node pools host application workloads. System pools should have at least 2-3 nodes for high availability and cannot be scaled down to zero. User pools allow independent scaling and different VM SKUs for cost optimization.
Separating concerns avoids noisy neighbors impacting control plane. For critical production, system pool uses Standard_B4ms while user pool may use spot instances.
Advanced Scenario: Your AKS cluster experiences sudden spikes in API server latency. How would you diagnose and mitigate?
Check API server load: kubectl top nodes, review API server metrics via Azure Monitor (API server latency, request rate). Causes: too many watchers, large etcd DB size. Mitigation: enable API server VNet integration, upgrade cluster tier, reduce informers, namespace quotas, --watch-cache tuning. Also review audit logs.
Senior approach: use Azure Advisor, analyze etcd metrics, use namespace segmentation for high cardinality objects.
Scenario Deploy a stateful Spring Boot app with persistent storage pinned to a specific Azure zone.
Use StatefulSet with Azure Disk – topology-aware scheduling via WaitForFirstConsumer. Define StorageClass with allowedTopologies to pin zones. PodAntiAffinity + volumeClaimTemplate. For max performance: Azure NetApp Files.
🗂️ ConfigMap & Secret (K8s)
Basic ConfigMap vs Secret differences + Spring Boot injection.
ConfigMap: plain text; Secret: base64 encoded. Injected as env vars or volumes. Spring Boot: @ConfigurationProperties / @Value, or spring.config.import=kubernetes:. Best practice: Azure Key Vault + Secrets Store CSI driver.
Advanced ConfigMap updated but pod still uses old values – dynamic reload without restart.
Use Stakater Reloader or rolling restart via hash annotation. Spring Cloud Kubernetes Config Watcher + @RefreshScope. Immutable releases: change ConfigMap → new deployment revision (hash) → rolling update.
Scenario Azure SQL credentials rotate every 30 days without pod restart.
Azure Key Vault Provider with rotationPollInterval. CSI driver syncs secret periodically. Or Azure AD Workload Identity + AAD authentication – eliminates rotation.
🌊 Istio Service Mesh
Basic How does Istio enable mTLS between services?
Istio injects Envoy sidecar proxies that establish mutual TLS using SPIFFE certificates. PeerAuthentication + DestinationRule enforce STRICT mTLS. Transparent to Spring Boot.
Advanced Canary release: shift 10% traffic to v2 using Istio VirtualService.
DestinationRule with subsets v1, v2. VirtualService: 90% v1, 10% v2 weight. Monitor metrics with Kiali/Prometheus, gradually increase weight. Traffic mirroring for testing.
route: - destination: host: order-service subset: v1 weight: 90 - destination: host: order-service subset: v2 weight: 10
Scenario Booking service random 5xx errors due to downstream timeouts – apply Istio resilience.
Retry and timeout in VirtualService: attempts:3 perTryTimeout:2s. Circuit breaker via OutlierDetection in DestinationRule. Combine with Spring Resilience4j.
🐘 Azure SQL Database
Basic Secure connection from AKS to Azure SQL without hardcoded credentials.
Azure AD Workload Identity + JDBC URL with Authentication=ActiveDirectoryManagedIdentity. Grant managed identity db_datareader, db_datawriter.
Advanced Connection timeouts under peak load (HikariCP) – diagnose & optimize.
Check HikariCP maxPoolSize, connectionTimeout. Monitor DTU/vCore limits. Use connectTimeout=60, socketTimeout=60. Enable accelerated networking. Spring Retry for transient faults. Enable leakDetectionThreshold.

Dedicated migration job (initContainer/K8s Job). Expand/contract pattern: add column, dual write, backfill, drop old column. Avoid long-locking ALTER TABLE.
🐳 Jib (Container Tools)
Basic Advantages of Jib over Dockerfile in CI/CD.
No Docker daemon, automatic layer separation (deps/resources/classes), faster push/pull, reproducible builds, distroless base. Great for Azure DevOps.
Advanced Jib + GraalVM native image for ultra-fast startup.
Jib focuses on JVM images. For native, use Spring Boot bootBuildImage (Paketo) or native-maven-plugin. Combine with AOT hints.
Scenario Slow CI builds due to dependency downloads – optimize with Jib.
Jib caches dependencies. In GitHub Actions, cache .m2/repository and Jib cache folder. jib:build pushes only changed layers to ACR, reducing CI time by 40-60%.

🧠 Java Design Patterns
Creational Singleton, Factory, Builder, Prototype.
Singleton: one instance (Runtime). Factory Method: subclass decides. Builder: step-by-step (StringBuilder). Prototype: clone.
Structural Adapter, Decorator, Proxy.
Adapter: convert interface; Decorator: add behaviour; Proxy: control access.
Key Comparisons Adapter vs Proxy vs Decorator, Strategy vs State.
Adapter changes interface; Proxy same interface (access control); Decorator same interface (adds behaviour). Strategy interchangeable algorithms (client sets); State changes with internal context.

🗂️ Java Memory & Garbage Collection (JDK7→17+)
Basic JVM memory areas & PermGen vs Metaspace.
Heap (young+old), stack, Metaspace (native memory since JDK8). PermGen fixed heap → OOM; Metaspace auto-grows.
JDK7: Young Gen → Old Gen → PermGen (fixed). JDK8+: Young + Old + Metaspace (native).
G1 GC layout, remember set, pause time goals.
G1 divides heap into regions (1-32 MB). Remembered Sets track cross-region references. SATB algorithm, pause prediction model, mixed GC. Default from JDK9+.
-Xlog:gc*:file=gc.log -XX:+HeapDumpOnOutOfMemoryError
ThreadLocal memory leak in web apps.
Always clean ThreadLocal with remove() in try-finally. Heap dump shows ThreadLocalMap references.

📡 Apache Kafka
Basic Partitions, offsets, consumer groups, at-least-once.
Partitions provide ordering, unique offsets. Consumer groups allow parallelism. At-least-once: commit after processing – duplicates possible. Exactly-once: idempotent producer + transactions + read_committed.
KRaft – ZooKeeper removal.
KRaft uses Raft consensus for metadata, eliminates external ZK, scales to millions of partitions. Migration: upgrade to 3.5+, metadata migrate, set process.roles=broker,controller.
kafka-storage.sh format --cluster-id <id> --standby
Consumer group rebalancing frequently – fix.
Long processing > max.poll.interval.ms. Increase interval, reduce max.poll.records, use CooperativeStickyAssignor, monitor GC pauses.
